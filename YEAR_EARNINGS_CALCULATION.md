## Cost of $MUND Payouts

### Redemption Scenarios

#### Scenario A: User redeems for subscription (most common)
- **73,192 $MUND → €73.19 subscription credit**
- **Developer's internal cost:** €73.19 margin on subscription
- **But:** User already paid €118.80, so next year's €118.80 is partially offset
- **Net impact:** User gets discount, but still pays €45.61 in year 2

#### Scenario B: User redeems for external vouchers
- **73,192 $MUND → €58.55 in partner vouchers**
- **Developer's cost:** €58.55 (you fulfill vouchers)
- **Developer keeps:** 20% margin = €58.55 × 0.20 = **€11.71 profit** on redemption

## Key Insights

1. **User Engagement ROI:** User gets back **61.6% of subscription cost** in Year 2 if fully engaged
2. **Developer Margin:** 48-64% profit margin (pre-tax) is healthy SaaS economics
3. **Breakeven:** Need ~3-5 active users to cover fixed costs (€200-300/month server, admin)
4. **Scalability:** As user base grows, affiliate revenue scales linearly, but $MUND liability grows too
5. **Retention Driver:** The 90-day airdrop (€10 value) creates strong retention incentive

---

## Risk Mitigation

### For Sustainability:
- 30-day escrow on cashback protects against returns/chargebacks
- External redemption has 20% margin buffer
- Subscription credits don't cost real money, just reduce future revenue
- Affiliate commission is real income that offsets $MUND payouts

### Red Flags to Watch:
- If >80% of users redeem externally every month
- If wearable fraud bypasses verification (fake step data)
- If affiliate partners reduce commission rates
- If ad CPM rates drop below €1.50

---

**Conclusion:** With this activity profile, you as developer earn **€49-65 net per active user annually** after all taxes. The user gets significant value back, creating a virtuous retention cycle. The model is sustainable if you maintain 60%+ profit margins and keep $MUND payouts controlled through escrow and verification systems.
